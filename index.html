01

What is CI/CD?

CI/CD stands for Continuous Integration and Continuous Deployment. It is a modern software development practice that automates the process of testing and deploying code. Continuous Integration ensures that every code change is automatically tested and merged into the shared repository, helping teams detect bugs early. Continuous Deployment takes it further by automatically releasing every validated change to production without manual intervention. Together, CI/CD allows development teams to ship software faster, more reliably, and with greater confidence — a core principle in cloud-native systems.

02

My Pipeline

After pushing code to the main branch, GitHub Actions automatically triggers the deploy.yml workflow. The pipeline first checks that all required files (index.html, style.css, script.js) exist, then uploads the site files as an artifact, and finally deploys them live to GitHub Pages — all without any manual steps.
